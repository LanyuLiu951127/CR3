## Requirement Specification (REQ)

### 1. 專題名稱 / Project Title / 프로젝트명

CR3 相同照片檢查系統 (CR3 Duplicate Photo Checker)

---

### 2. 專案目的 / Purpose / 목적

攝影師在拍攝後，會從相機 SD 卡中匯出大量 CR3 檔案（Canon RAW 格式）。這些檔案可能包含多張幾乎相同的照片，造成挑選效率低下。
此系統的目的在於：

* 自動檢查資料夾內有無相同或重複的 CR3 檔案。
* 讓攝影師能更快速瀏覽相同照片群組，方便挑選最佳版本。
* 提供清晰、直覺的前端操作介面。

---

### 3. 主要使用者 / Target Users / 주요 사용자

* 攝影師：進行拍攝後的檔案整理與檢查。
* 攝影助理或後製人員：在篩選、分類照片前先進行重複檢查。

---

### 4. 網站主要功能 / Key Features / 주요 기능

| 分類   | 功能描述                         | 優先級    |
| ---- | ---------------------------- | ------ |
| 首頁   | 顯示系統名稱與簡介，提供「選擇資料夾」與「開始檢查」按鈕 | Must   |
| 檢查頁  | 顯示檢查進度提示（例如「正在分析照片中...」）     | Must   |
| 結果頁  | 顯示重複照片群組，讓使用者瀏覽相同或相似的照片      | Must   |
| 返回首頁 | 提供重新選擇資料夾的按鈕                 | Should |
| 使用說明 | 提供簡短說明：支援檔案類型、操作方式、注意事項      | Could  |

---

### 5. 成功指標 / Success Metrics / 성공 지표

* 系統可成功顯示資料夾中所有 CR3 檔案。
* 成功找出並清楚呈現相同照片的組合。
* 使用者能夠清楚分辨每組相同照片。
* 網頁在電腦瀏覽器上顯示正常，排版整齊且直覺。

---

### 6. 非範圍內容 / Out of Scope / 범위 밖

* 不包含實際圖像比對演算法（由後端或其他模組處理）。
* 不包含登入系統或使用者權限設定。
* 不包含照片上傳、下載或雲端同步功能。
* 不開發手機 App 版本。

---

### 7. 前端介面設計 / Frontend Structure（HTML + CSS 架構，不含 JS）

#### (1) 首頁設計

**目的：** 作為進入系統的入口，提供使用者基本操作。
**內容：**

* 系統名稱（頂端置中）
* 簡短文字說明（說明系統用途與操作方式）
* 「選擇資料夾」按鈕（用於選擇 CR3 檔案所在資料夾）
* 「開始檢查」按鈕（用於啟動比對流程）

**版面建議：**

* 整體版面置中，採用簡潔風格。
* 兩個主要按鈕應明顯、易於點擊。
* 可以在按鈕下方加入一段小提示文字（例如「請先選擇資料夾再開始檢查」）。

---

#### (2) 檢查進度畫面

**目的：** 在比對過程中顯示狀態，避免使用者誤以為系統卡住。
**內容：**

* 提示文字：「正在檢查中，請稍候...」
* 可搭配簡易的進度條或動畫效果（純 CSS 呈現即可）。
* 背景可維持簡潔風格，使注意力集中在進度上。

---

#### (3) 結果顯示頁

**目的：** 呈現比對結果，讓使用者快速找到相同照片。
**內容：**

* 標題：「比對結果」
* 每組重複照片以「群組」方式呈現：

    * 群組標題：「第 X 組相同照片」
    * 該組底下顯示 2 張或以上相同照片的縮圖。
* 若無相同照片，顯示提示：「未發現相同照片」。

**排版建議：**

* 每組以區塊方式分隔，背景淺灰或白色，邊界清楚。
* 每組內照片橫向排列，間距一致。
* 可在照片下方保留檔名或編號文字區。

---

#### (4) 頁尾區（Footer）

**內容：**

* 版權資訊（例如「© 2025 CR3 Checker」）
* 「返回首頁」按鈕（可讓使用者重新選擇資料夾進行新一輪檢查）

**設計建議：**

* 固定於頁面底部或置中。
* 字體小於主內容，以淺色呈現。

---

### 8. 頁面結構摘要

| 頁面名稱 | 主要內容                         | 說明         |
| ---- | ---------------------------- | ---------- |
| 首頁   | 系統名稱、說明文字、「選擇資料夾」按鈕、「開始檢查」按鈕 | 系統進入點      |
| 檢查中頁 | 提示文字、進度條動畫                   | 告知使用者系統運作中 |
| 結果頁  | 群組顯示相同照片、檔案資訊、「返回首頁」按鈕       | 呈現分析結果     |

---

### 9. 整體設計風格建議

* 色系：以白、灰、淺藍為主，營造乾淨、專業感。
* 字型：使用簡潔的無襯線字體（如 Noto Sans、Arial）。
* 對齊方式：採置中或左對齊，確保圖片與文字整齊排列。
* 響應式設計：可於大螢幕與筆電正常顯示，手機版本可後續再開發。
