🧩 系統架構層面（SA 問題)
這個系統的主要「任務」是什麼？ 이 시스템의 주요 "임무"는 무엇인가요?
如果你是使用者，會先看到什麼畫面？接著會做什麼？ 사용자라면 어떤 화면을 먼저 보게 되나요? 다음으로 무엇을 할 건가요?
系統中有哪些資料需要被保存？由誰管理？ 시스템에 어떤 데이터가 저장되어야 하나요? 누가 관리하나요?
哪些模組之間需要交換資料？ 어떤 모듈들 사이에 데이터 교환이 필요한가요?
如果要讓系統更自動化，哪一部分最值得優化？ 시스템을 더 자동화하려면 어느 부분이 가장 최적화할 가치가 있나요?
哪一個模組最重要？如果拿掉它會發生什麼？ 가장 중요한 모듈은 무엇인가요? 그것을 제거하면 어떤 일이 발생하나요?

System Architecture (SA) 範例
1. 系統概覽 / Overview / 개요
   本網站分為三層結構:
   前端 (HTML, CSS, JS):顯示內容與使用者互動
   後端 (Spring Boot):處理請求與資料
   資料層 (Database):儲存公告與活動資訊
2. 模組設計 / Module Design / 모듈 설계
   模組
   功能
   負責人

   儲存公告與活動資料

4. 資料流 / Data Flow / 데이터 흐름
   使用者 → 前端頁面 → 後端 API → 資料庫 → 回傳結果
5. 非功能性需求 / Non-functional Req / 비기능 요구사항
   效能:同時支援 100 名使用者
   安全性:基本輸入驗證與錯誤處理
   擴充性:模組化架構,支援未來新增功能
