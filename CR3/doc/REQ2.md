# 專案需求規格 (Requirement Specification) - v2

## 1. 專案名稱 / Project Title

CR3 相同照片檢查系統 (CR3 Duplicate Photo Checker)

---

## 2. 專案目的 / Purpose

本專案旨在開發一個桌面應用程式，解決攝影師在整理 Canon CR3 格式相片時，因存在大量重複或相似照片而導致效率低落的問題。透過自動化檢查，幫助使用者快速找出相同照片群組，以便進行篩選與管理。此專案將作為 Python 期末專題，展現實務開發能力。

---

## 3. 主要使用者 / Target Users

*   攝影師：拍攝後需要整理大量 CR3 檔案的專業人士。
*   攝影愛好者：希望有效管理個人攝影作品集的使用者。

---

## 4. 應用程式主要功能 / Key Features

| 分類     | 功能描述                                                                                                      | 優先級 |
| -------- | ------------------------------------------------------------------------------------------------------------- | ------ |
| **核心功能** | **選擇資料夾**：提供圖形化介面按鈕，讓使用者能選擇本機存放 CR3 檔案的資料夾。                                         | Must   |
| **核心功能** | **啟動檢查**：使用者選擇資料夾後，點擊按鈕開始分析。程式應在背景遍歷檔案，找出重複的照片。                           | Must   |
| **核心功能** | **儲存與讀取分析結果**：為了加速二次檢查，程式應將照片的特徵值 (例如 Hash) 存成 `.json` 或 `.csv` 檔案。下次檢查時可直接讀取，避免重複計算。 | Must   |
| **介面呈現** | **顯示進度**：在檢查過程中，介面應顯示「正在分析中...」的提示或進度條，告知使用者系統正在運作。                     | Must   |
| **介面呈現** | **結果分組顯示**：以群組方式呈現所有重複的照片。每個群組應包含至少 2 張相同照片的縮圖與檔案名稱。                      | Must   |
| **介面呈現** | **無結果提示**：若資料夾內沒有任何重複的照片，應在介面顯示「未發現相同照片」的訊息。                                 | Should |
| **輔助功能** | **重新開始**：提供「返回」或「重新選擇」按鈕，讓使用者能方便地檢查另一個資料夾。                                     | Should |
| **輔助功能** | **使用說明**：在介面中提供簡短的文字說明，指導使用者如何操作。                                                       | Could  |

## 5. 使用技術與架構 / Technology Stack

*   **程式語言**: Python 3
*   **GUI 框架**: Tkinter 或 PySimpleGUI (用於建構桌面應用程式介面)
*   **資料處理**:
    *   使用 `list` 與 `dict` 管理照片資料與重複群組。
    *   使用 `json` 或 `csv` 模組進行分析結果的儲存與讀取。
*   **圖像處理**: Pillow (PIL Fork) 函式庫 (用於讀取圖片檔案、生成縮圖)。
*   **檔案系統**: `os` 與 `glob` 模組 (用於遍歷資料夾與檔案)。

---

## 6. 成功指標 / Success Metrics

*   程式能成功啟動，並顯示圖形化操作介面。
*   使用者能透過介面成功選擇資料夾，並啟動分析程序。
*   程式能正確找出重複的 CR3 檔案，並以分組方式顯示結果。
*   程式能生成分析記錄檔 (`.json`/`.csv`)，並在下次啟動時利用此檔案加速分析。
*   介面操作流暢，排版整齊，符合直覺。

---

## 7. 非範圍內容 / Out of Scope

*   不支援 CR3 以外的 RAW 檔案格式 (例如 NEF, ARW)。
*   不提供線上雲端同步或上傳/下載功能。
*   不包含使用者登入或權限管理系統。
*   不進行複雜的「相似」圖像比對 (例如比較構圖、顏色)，僅專注於檔案層級或 Hash 值的「相同」比對。
